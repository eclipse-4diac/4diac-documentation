In this tutorial you create a simple Blinking application -- the "Hello World" of automation programs.
It teaches the key workflow of 4diac IDE.
The application runs locally, so no external hardware or PLC is needed.

== [[createSystem]]Create a new IEC 61499 System

. Open the System Perspective: click the system icon in the toolbar.
. In the top menu bar, go to: [.menu4diac]#File → New → 4diac IDE Project...#
. In the wizard, name your project _BlinkyTest_ and click Finish.
+
image:Step1/blinkyTest_project_create.png[Create a new IEC 61499 System]

After creating the project, the [.view4diac]#System Explorer# on the left shows the project _BlinkyTest_ with the following items:

* *BlinkyTest []* -- the system file; expand it to find _BlinkyTestApp_ (your application) and _System Configuration_ (where you define your hardware/devices)
* *Type Library* -- the Function Block types available in this project
* *External Libraries* -- shared type libraries
* *Standard Libraries* -- the standard 4diac type libraries
* *MANIFEST.MF* -- the Eclipse project manifest

image:Step1/SystemExplorer_left.png[System Explorer showing project structure]

== [[createNetwork]]Create the FB Network (Blink Application)

Double-click _BlinkyTestApp_ in the [.view4diac]#System Explorer# to open the Application Editor.
Then drag and drop the following three Function Blocks from the palette (look in the [.folderLocation]#events# folder):

* `E_CYCLE`
* `E_SWITCH`
* `E_SR`

You can also double-click an empty area in the editor and search for them by name.
The name in the middle of a block is its type; the name on top is the instance name (must be unique).

image:Step1/fbs_in_editor.png[Function Blocks in the Application Editor]

=== Connect the FBs

Connect them by hovering over an output pin, clicking, and dragging to the input pin of the next block.

* Green lines = Event connections (top part of FB)
* Purple lines = Data connections (bottom part of FB)

image:Step1/connected_blocks.png[Connected Function Blocks]

=== Set the Cycle Time (DT Input)

The `E_CYCLE` block needs a cycle time.
Set DT to `T#1s` (1 second) using one of these methods:

* Double-click the DT on the left buttom of the `E_CYCLE` block
* Select DT and change the value in the Properties tab below

image:Step1/set_dt.png[Setting the DT value on E_CYCLE]

NOTE: DT must be set, otherwise `E_CYCLE` does not know how often to fire.
`T#1s` means "every 1 second".

The blink works as follows: when a `START` event arrives at `E_CYCLE.START`, every second `E_CYCLE.EO` fires.
`E_SWITCH` routes the event to `EO0` when `G=0`, or `EO1` when `G=1`.
`E_SR` sets its output `Q` to `TRUE` on `S` and `FALSE` on `R`.
`Q` feeds back to `G`, so `Q` toggles `TRUE`/`FALSE` every second -- blink!

== [[configureHardware]]Configure the Hardware (System Configuration)

Double-click [.view4diac]#System Configuration# in the [.view4diac]#System Explorer# to open the System Configuration Editor.
Use drag-and-drop from the palette to add:

. `FORTE_PC` (a Device) -- represents your local computer
. `Ethernet` (a Segment) -- the communication medium

Then connect them by dragging from the Device to the Segment.
The device comes with one `EMB_RES` resource already.

Rename the device to _testee_ (double-click the name or use Properties below).
Check that `MGR_ID` is `localhost:61499` (or similar port).
Check that the Profile is set to `HOLOBLOC` in the Properties tab.

image:Step1/system_configuration.png[System Configuration with device renamed and properties configured]

Rename the `EMB_RES` resource to _Blinky_RES_ (double-click the `EMB_RES`).
`EMB_RES` stands for Embedded Resource -- it's where your FBs will actually run.

== [[mapNetworks]]Map FBs to the Device/Resource

Go back to the Application Editor (_BlinkyTestApp_).
Select all three FBs (click + drag or Ctrl+A), then:

[.menu4diac]#right click → Map To... → testee → Blinky_RES#

image:Step1/map_to.png[Mapping FBs to the resource]

After mapping, the FBs change color to match the device color.
To unmap, right-click a FB → [.menu4diac]#Unmap#.

== [[configureResource]]Configure the Resource (Add START)

Double-click _Blinky_RES_ in the [.view4diac]#System Explorer# (or in System Configuration) to open the Resource Editor.
You'll see:

* Your mapped FBs (`E_CYCLE`, `E_SWITCH`, `E_SR`)
* A white `START` block -- fires a `COLD` event when the PLC starts

image:Step1/what_is_this.png[Resource Editor showing the START block and mapped FBs]

Connect both `COLD` and `WARM` outputs of the `START` block to `E_CYCLE.START`.
This is the starting event that kicks off the whole blink loop.

== [[deployingToForte]]Deploy to 4diac FORTE

Before deploying, point 4diac IDE to the FORTE executable.
Go to: [.menu4diac]#4diac IDE → Settings → FORTE Preferences#

In [.addressDoc]#FORTE Location#, browse to your `forte` (or `forte.exe` on Windows) file.
Click _Apply and Close_.

image:Step1/select_4diac_forte2.png[Selecting the FORTE executable,width=600]

. Start 4diac FORTE (run the executable locally)
. In 4diac IDE, click the green Run/Deploy arrow in the toolbar
. The Console should show:
+
[source,console]
----
<!-- Connected to device: testee -->
----

To redeploy, right-click the Blinky_RES in [.view4diac]#System Explorer# → [.menu4diac]#Delete Resource# to wipe it clean, then deploy again.

image:Step1/delete_resource.png[Delete Resource option in the context menu]

== [[monitoringApplication]]Monitor the Application

Once the application is deployed, you can watch it live directly in the Resource Editor.
Open _Blinky_RES_ and right-click on any connection or pin (e.g. the `Q` output of `E_SR`), then select [.menu4diac]#Toggle Watch# from the context menu.

image:Step1/toggle_watch_variable.png[Toggle Watch on a connection in the Resource Editor]

The watched values appear in the Watch panel on the right.
You'll see `Q` alternating between `true` and `false` every second -- your light is blinking!
You can also trigger events manually by right-clicking on a pin and selecting [.menu4diac]#Trigger Event#.
